# 游戏抢单系统 GUI 使用说明

## 启动GUI

有两种方式启动GUI界面：

### 方式1：使用命令行参数
```bash
python main.py --gui
```

### 方式2：直接运行GUI模块
```bash
python gui.py
```

## 功能说明

### 1. Cookie设置
- **加载**：从 `cookie.txt` 文件加载已保存的Cookie
- **保存**：将输入的Cookie保存到 `cookie.txt` 文件
- **从文件导入**：从其他文件导入Cookie

### 2. 游戏搜索
- 在"游戏名"输入框中输入要搜索的游戏名称
- 点击"搜索"按钮或按回车键开始搜索
- 搜索结果会显示在列表中，包括：
  - 游戏名称
  - 价格
  - 购买状态

### 3. 选择游戏
- **点击列表项**：点击游戏列表中的任意位置可以切换选择状态（☐/☑）
- **全选**：点击"全选"按钮选择所有游戏
- **取消全选**：点击"取消全选"按钮取消所有选择
- **清空列表**：清空当前搜索结果

### 4. 开始抢单
- 选择要抢单的游戏后，点击"开始抢单"按钮
- 系统会开始监控选中的游戏，当游戏可购买时自动尝试抢单
- 抢单过程会实时显示在日志区域
- 点击"停止抢单"按钮可以随时停止

### 5. 运行日志
- 所有操作和抢单状态都会显示在底部的日志区域
- 日志包括时间戳和日志级别
- 可以滚动查看历史日志

## 注意事项

1. **Cookie设置**：使用前必须先设置有效的Cookie，否则无法搜索和抢单
2. **网络连接**：确保网络连接正常，系统需要访问游戏API
3. **线程安全**：GUI使用多线程处理搜索和抢单，避免界面卡顿
4. **数据保存**：Cookie会自动保存到 `cookie.txt` 文件，下次启动会自动加载

## 故障排除

- **GUI无法启动**：确保已安装 tkinter（Python标准库，通常已包含）
- **搜索无结果**：检查Cookie是否有效，网络是否正常
- **抢单无响应**：检查API地址是否正确配置（需要修改 `game_searcher.py` 和 `order_grabber.py` 中的URL）
